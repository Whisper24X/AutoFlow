# 闭环 Demo 需求说明

## 背景
我们要在 Cursor IDE 中演示一条完整工程闭环：
从需求出发，完成开发，运行自动化测试，发现并修复缺陷，再回归验证通过。

## 业务需求
- 页面提供订单总价计算器，输入项包含：
  - 单价 `price`
  - 数量 `quantity`
  - 优惠码 `coupon`
- 当优惠码是 `SAVE10` 时，总价应按 9 折计算。
- 其他优惠码不打折。

## 验收标准
- `price=100`、`quantity=2`、`coupon=SAVE10` 时，页面显示 `¥180.00`。
- 自动化测试必须覆盖上述关键路径。
- 测试应能先发现缺陷，再在修复后通过，形成闭环证据。
